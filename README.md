👥 System Roles & Features
1. 👤 User (Sender)
Feedback Submission: Submit suggestions, complaints, or inquiries through an intuitive form.

Category Filtering: Tag feedback based on type (e.g., Bug Report, Feature Request, General Feedback).

Status Tracking: View the current status of submitted feedback (e.g., Pending, Under Review, Resolved).

2. 🛠️ Admin (System Manager)
Feedback Dashboard: Centralized view of all incoming feedback with real-time counters for pending vs. resolved items.

Management & Response: Ability to view, update, and provide official responses to user submissions.

CRUD Operations: Manage feedback categories and system configurations.

User Management: Oversee user account status and ensure proper documentation of reports.

🛠️ Tech Stack Used
Frontend: HTML5, CSS3 (Responsive Design with Flexbox/Grid), JavaScript (Dynamic Form Handling & Modals)

Backend: PHP (Core Logic & Request Handling)

Database: MySQL / MariaDB (Relational design, optimized for query performance and data integrity)

Styling Font: Poppins / Segoe UI

🚀 Installation & Local Setup
Prerequisites
Download and install XAMPP or any local PHP server environment.

Steps
Set up the Database:

Open XAMPP Control Panel and start Apache and MySQL.

Navigate to http://localhost/phpmyadmin/.

Create a new database (e.g., feedback_db), click on the SQL tab, paste your database schema script, and click Go.

Run the Project:

Clone this repository into your htdocs folder (e.g., C:/xampp/htdocs/FeedbackSys/).

Configure your database connection in db_connect.php (or your config file).

Open your browser and navigate to:

http://localhost/FeedbackSys/index.php
